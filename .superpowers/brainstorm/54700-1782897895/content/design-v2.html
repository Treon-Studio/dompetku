Desain Fitur: Rekening Saya (Payment Accounts)

1. Model Data (SQLite / Drizzle)

export const payment_accounts = sqliteTable("payment_accounts", {
  id: text("id").primaryKey().$defaultFn(() => crypto.randomUUID()),
  bank_name: text("bank_name").notNull(),
  account_number: text("account_number").notNull(),
  account_holder: text("account_holder").notNull(),
  qris_image: text("qris_image"),
  user_id: text("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  created_at: text("created_at").default(sql`CURRENT_TIMESTAMP`),
  updated_at: text("updated_at").default(sql`CURRENT_TIMESTAMP`),
});

2. Alur Pengguna (User Flow)

  • User mengklik menu "Rekening Saya" di Sidebar.
  • Halaman menampilkan daftar rekening dalam bentuk kartu/list dengan ikon bank/ewallet.
  • Setiap kartu rekening memiliki tombol "Salin Nomor" (dengan toast sukses) dan tombol "Lihat QRIS" (membuka dialog gambar QRIS).
  • Tombol "Tambah Rekening" membuka form input (Nama Bank/Dompet, Nomor, Nama Pemilik, dan Unggah File QRIS opsional).
  • User bisa mengedit atau menghapus rekening.

3. Integrasi dengan Halaman Share (Hutang)

  • Pada halaman /share/$slug (halaman publik yang dilihat teman), jika teman masih memiliki sisa hutang (UNPAID) kepada pemilik akun, tampilkan kartu "Cara Pembayaran" di bawah nominal sisa hutang.
  • Kartu ini berisi daftar rekening pemilik akun (yang sebelumnya dibuat dari fitur Rekening Saya) dengan:
    • Ikon Bank/Dompet, Nama Bank/Dompet, Nama Pemilik, dan Nomor Rekening.
    • Tombol "Salin" di samping nomor rekening (sama dengan toast sukses di halaman internal).
    • Tombol "Lihat QRIS" jika gambar QRIS tersedia, membuka modal/dialog dengan QR code untuk di-scan.
  • UX: Kartu pembayaran muncul dengan jelas, hanya ditampilkan ketika ada sisa hutang, dan menarik secara visual agar teman tahu ke mana harus membayar.
